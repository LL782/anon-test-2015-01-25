Tabular Lists - Pattern demo

Demonstration of the tabular lists pattern

Summary

This pattern provides a table-like component made up off a group of lists. This is to allow the table to flow into multiple lines for an attractive view on a narrow viewport. A data attribute data-vis-cols is given a value equal to the number of columns in the table. Using this variable CSS rules define how the component should be laid out at different breakpoints.

e.g. on a narrow view port a 16 column table is laid out as 4 consecuitive, 4 column tables so that each column has a comfortable width.

Markup pattern

Zen coding markup pattern:
.tabular-list-wrap[data-vis-cols="n"]>ol.tabular*n>li.list-header+li

The wrap elemnt has class tabular-list-wrap and an attribute data-vis-cols. Inside of this are a number of ordered-lists equal to the number of columns in the table. Each list has the class tabular. The first list-item has the class list-header. There should be an equal number of list items in all lists within the same tabular-list-wrap.

Overflow and mutliples

If a counter needs to be include with a list item that list item should use the has-multiple class and the counter should be a span with a class of multiple-counter. This will ensure that the counter is always visible even if the text overflows the container

The prevent overflowing text from distrupting the layout the text-overflow:ellispis applies to the list-items. The list items are given a title attribute with the entire text content of the item so that users can access the full information if it has overflowed.
